# Lark 技术架构文档

> **定位**：Windows桌面效率工具，极简主义鼠标矩阵面板
> **目标**：提供快速文件搜索、应用启动、插件扩展的统一入口
> **技术理念**：轻量级、响应优先、松耦合

---

## 一、核心架构

### 1.1 整体架构设计原则

**架构模式选择理由：**
- **事件驱动**：桌面工具的核心交互特点是异步触发（热键、搜索输入、鼠标操作），事件驱动模式天然匹配这种场景
- **模块化**：工具功能独立且可扩展（搜索、应用、插件），模块化架构便于独立开发和演进
- **松耦合通信**：通过事件信号解耦模块依赖，支持热插拔式功能扩展

**整体架构：**

```
┌──────────────────────────────────────────────────────────┐
│                     主控制器                               │
│            (协调器 + 事件分发中心)                         │
├──────────────────────────────────────────────────────────┤
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌────────┐  │
│  │热键监听器│  │窗口管理器│  │搜索引擎  │  │插件系统│  │
│  └──────────┘  └──────────┘  └──────────┘  └────────┘  │
│       │             │             │             │        │
└───────┼─────────────┼─────────────┼─────────────┼────────┘
        │             │             │             │
        ▼             ▼             ▼             ▼
  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌────────┐  │
  │配置管理器│  │应用管理器│  │配置管理器│  │配置存储│  │
  └──────────┘  └──────────┘  └──────────┘  └────────┘  │
```

### 1.2 系统分层设计

#### 应用层（Application Layer）
**职责**：用户交互入口和界面呈现

**设计要点：**
- **工具面板**：核心交互区域，采用悬浮窗口设计
  - 搜索模块：实时搜索输入框 + 结果列表
  - 应用模块：快捷方式图标网格，末尾固定"+"新增按钮
  - 设置模块：配置项分类管理（热键、应用、插件、界面）
- **设置界面**：独立的配置窗口，支持实时预览
- **插件管理界面**：插件启用/禁用、插件信息展示

#### 业务逻辑层（Business Logic Layer）
**职责**：核心功能逻辑处理

**设计要点：**
- **热键管理**：全局热键监听、热键冲突检测
- **窗口管理**：窗口生命周期、位置计算、动画效果
- **搜索管理**：搜索请求构建、结果解析、分页处理
- **应用管理**：快捷方式增删改查、使用统计
- **插件管理**：插件发现、加载、注册、生命周期管理

#### 数据访问层（Data Access Layer）
**职责**：数据持久化和访问

**设计要点：**
- **配置存储**：JSON格式，便于手动编辑和备份
- **应用数据**：快捷方式元信息（名称、路径、图标、使用次数）
- **插件数据**：插件清单、启用状态、配置项

## 二、核心模块设计

### 2.1 主控制器（Main Controller）

**定位**：系统协调器和事件总线

**核心职责：**
1. 生命周期管理：启动初始化、运行时协调、关闭清理
2. 事件路由：接收各模块信号，分发给目标模块
3. 异常处理：统一异常捕获和恢复机制
4. 模块依赖管理：控制模块初始化顺序

**关键设计决策：**
- 采用单例模式，确保全局唯一协调点
- 实现事件注册表，支持动态订阅/取消订阅
- 提供模块健康检查接口，便于故障隔离

### 2.2 热键监听器（Hotkey Listener）

**定位**：用户输入的入口拦截器

**核心职责：**
1. 全局热键注册和监听
2. 热键冲突检测和提示
3. 支持多热键配置（主热键+备用热键）
4. 热键触发事件的标准化输出

**关键设计决策：**
- 支持键盘热键和鼠标侧键（默认XButton1）
- 热键配置支持从配置文件动态热重载
- 实现热键响应防抖机制，避免重复触发

**技术约束：**
- 热键监听需要在系统级别注册，需要管理员权限可能
- 不同输入法环境下热键兼容性考虑

### 2.3 窗口管理器（Window Manager）

**定位**：界面呈现和交互控制器

**核心职责：**
1. 悬浮窗口的创建和销毁
2. 窗口位置计算（跟随鼠标、智能避让）
3. 窗口动画效果（弹出/收起过渡）
4. 窗口拖动和交互事件处理

**关键设计决策：**
- 窗口特性：无边框、半透明、圆角、始终置顶
- 位置计算策略：
  - 鼠标右侧优先，超出屏幕时自动调整
  - 避免遮挡鼠标光标和关键操作区域
- 动画效果：
  - 弹出：淡入+位移动画，时长控制在200ms内
  - 收起：淡出动画，时长控制在150ms内
- 窗口拖动：按住标题栏或空白区域拖动，释放时吸附屏幕边缘

**性能考量：**
- 窗口显示/隐藏使用显隐切换而非重建，减少资源消耗
- 动画使用GPU加速，避免CPU占用过高

### 2.4 搜索引擎（Search Engine）

**定位**：快速文件检索的核心引擎

**核心职责：**
1. 调用Everything命令行工具执行搜索
2. 构建搜索参数和解析搜索结果
3. 实现实时搜索（防抖优化）
4. 搜索结果分页和排序

**关键设计决策：**
- 集成方式：直接调用项目内置es.exe，不依赖系统安装
- 搜索参数：
  - 支持通配符搜索（*.txt）
  - 支持路径过滤（指定目录）
  - 支持文件类型过滤（-ext参数）
  - 支持结果数量限制（默认50条）
- 实时搜索：
  - 输入防抖：300ms无输入后触发搜索
  - 增量搜索：基于上次结果进一步筛选（可选）
- 结果解析：
  - 优先解析JSON输出（-json参数）
  - 降级支持文本输出解析

**性能优化：**
- 搜索结果缓存（相同关键词1分钟内复用）
- 异步搜索，避免阻塞UI线程
- 搜索取消机制（用户连续输入时取消上一次搜索）

**数据格式要求：**
- 搜索结果需包含：文件名、完整路径、文件大小、修改时间、文件类型

### 2.5 应用管理器（Application Manager）

**定位**：应用快捷方式的统一管理中心

**核心职责：**
1. 应用快捷方式的增删改查
2. 应用列表的排序和分组
3. 应用启动和错误处理
4. 应用使用统计和分析

**关键设计决策：**
- 数据结构：
  - 应用对象包含：唯一ID、名称、路径、图标路径、使用次数、创建时间
  - 支持分组：应用可分配到不同分类（工作、娱乐、工具等）
- 界面呈现：
  - 网格布局，自适应窗口大小
  - 末尾固定"+"按钮，点击弹出应用选择对话框
  - 支持拖拽排序和拖拽分组
- 应用启动：
  - 路径有效性检查
  - 启动参数支持（可配置启动参数）
  - 启动失败提示和日志记录
- 使用统计：
  - 记录每次启动时间和使用时长
  - 支持按使用频率自动排序

**扩展性考虑：**
- 支持从开始菜单、桌面、常用文件夹批量导入应用
- 支持快捷方式关联（一个应用多个入口）

### 2.6 插件系统（Plugin System）

**定位**：功能扩展的开放平台

**核心职责：**
1. 插件发现和加载
2. 插件生命周期管理
3. 插件间通信和依赖管理
4. 插件权限控制

**关键设计决策：**
- 插件接口设计：
  - 必须实现：get_icon()（返回图标）、handle_click()（处理点击事件）、get_info()（返回插件信息）
  - 可选实现：on_load()（加载时回调）、on_unload()（卸载时回调）、get_settings()（返回配置界面）
- 插件发现：
  - 扫描plugins/目录下的.py文件
  - 识别符合接口规范的类
  - 读取插件元信息（名称、版本、作者、描述）
- 插件隔离：
  - 每个插件在独立命名空间运行
  - 插件异常不影响主程序
  - 插件超时控制（避免插件卡死）
- 插件通信：
  - 通过事件总线通信，避免直接依赖
  - 支持插件间数据共享（通过配置管理器）

**权限模型：**
- 插件默认无系统权限
- 敏感操作需要用户授权（文件读写、网络访问）
- 插件沙箱机制（可选）

**扩展性考虑：**
- 支持插件依赖声明（版本、依赖库）
- 支持插件热加载（无需重启程序）
- 预留插件市场接口（未来扩展）

### 2.7 配置管理器（Configuration Manager）

**定位**：配置数据的统一存储和访问入口

**核心职责：**
1. 配置文件的读写和管理
2. 配置项的验证和默认值处理
3. 配置变更通知和热重载
4. 配置备份和恢复

**关键设计决策：**
- 存储格式：
  - 主配置文件：config/settings.json
  - 应用数据：config/apps.json
  - 插件配置：config/plugins.json
- 配置结构：
  - 热键配置：trigger_key、fallback_keys
  - 窗口配置：width、height、opacity、corner_radius
  - 搜索配置：max_results、search_paths、exclude_paths
  - 应用配置：app_list、groups、auto_sort
  - 插件配置：enabled_plugins、plugin_settings
- 配置验证：
  - 配置项类型检查（数字、字符串、布尔值）
  - 配置项范围验证（透明度0-100）
  - 配置项依赖验证（热键冲突检测）
- 配置热重载：
  - 监听配置文件变化（基于文件系统监控）
  - 配置变更时通知相关模块更新
  - 支持配置回滚（配置错误时恢复上一次有效配置）

**扩展性考虑：**
- 支持配置导入/导出（便于用户分享配置）
- 支持多用户配置（不同用户使用不同配置文件）
- 支持云同步（预留接口）

## 三、关键数据流

### 3.1 工具面板打开流程

**触发条件：** 用户按下定义的热键（默认鼠标侧键XButton1）

**流程步骤：**
1. 热键监听器捕获热键事件
2. 热键监听器验证热键有效性（检查是否被其他程序占用）
3. 热键监听器发送"show_window"事件给主控制器
4. 主控制器验证窗口状态（避免重复显示）
5. 主控制器通知窗口管理器显示窗口
6. 窗口管理器计算窗口最佳位置（基于鼠标位置和屏幕边界）
7. 窗口管理器创建/显示窗口（使用已存在的窗口实例）
8. 窗口管理器触发"window_shown"事件
9. 主控制器通知搜索引擎和应用管理器初始化界面
10. 搜索引擎加载搜索历史（可选）
11. 应用管理器加载应用列表（根据配置排序）

**异常处理：**
- 热键冲突：提示用户并建议更换热键
- 窗口创建失败：记录错误日志，提示用户重启程序
- 配置加载失败：使用默认配置，记录错误日志

### 3.2 搜索功能数据流

**触发条件：** 用户在搜索框输入关键词

**流程步骤：**
1. 搜索框组件捕获输入事件
2. 搜索框组件触发防抖定时器（300ms）
3. 防抖到期后，搜索框发送"search_request"事件（包含关键词）
4. 主控制器接收事件，转发给搜索引擎
5. 搜索引擎检查缓存（相同关键词是否存在缓存）
6. 缓存未命中，搜索引擎构建命令行参数
7. 搜索引擎异步调用es.exe执行搜索
8. es.exe返回搜索结果（JSON格式）
9. 搜索引擎解析JSON，提取文件信息
10. 搜索引擎发送"search_result"事件（包含结果列表）
11. 主控制器接收事件，转发给搜索模块
12. 搜索模块更新界面，显示搜索结果
13. 搜索模块实现分页（每页显示10-20条）

**优化措施：**
- 输入防抖：避免频繁触发搜索
- 结果缓存：减少重复搜索
- 异步执行：不阻塞UI线程
- 搜索取消：用户连续输入时取消上一次搜索

### 3.3 应用启动数据流

**触发条件：** 用户点击应用图标

**流程步骤：**
1. 应用模块捕获点击事件
2. 应用模块验证应用路径有效性
3. 应用模块发送"launch_app"事件（包含应用信息）
4. 主控制器接收事件，转发给应用管理器
5. 应用管理器检查应用是否已在运行（可选）
6. 应用管理器构建启动命令（包含启动参数）
7. 应用管理器调用系统API启动应用
8. 应用管理器记录启动时间和使用次数
9. 应用管理器发送"app_launched"事件
10. 主控制器更新使用统计（可选）
11. 窗口管理器自动隐藏窗口（可选配置）

**异常处理：**
- 路径无效：提示用户重新配置应用路径
- 启动失败：记录错误日志，提示用户检查应用权限
- 应用无响应：提供强制结束选项

### 3.4 插件使用数据流

**触发条件：** 用户启用插件并点击插件图标

**流程步骤：**
1. 设置界面捕获插件启用事件
2. 设置界面发送"plugin_enable"事件（包含插件名称）
3. 主控制器接收事件，转发给插件系统
4. 插件系统检查插件依赖是否满足
5. 插件系统加载插件模块
6. 插件系统调用插件的on_load()方法（如果实现）
7. 插件系统注册插件接口
8. 插件系统发送"plugin_loaded"事件
9. 主控制器通知应用管理器在界面中添加插件图标
10. 用户点击插件图标
11. 应用模块发送"plugin_click"事件（包含插件实例）
12. 主控制器接收事件，转发给插件系统
13. 插件系统调用插件的handle_click()方法
14. 插件执行具体功能
15. 插件发送"plugin_result"事件（如果需要返回结果）

**异常处理：**
- 插件加载失败：记录错误日志，禁用插件
- 插件执行超时：强制终止插件，提示用户
- 插件异常：捕获异常，不影响主程序

### 3.5 配置保存流程

**触发条件：** 用户修改设置或应用列表发生变化

**流程步骤：**
1. 设置界面捕获配置变更事件
2. 设置界面验证配置有效性
3. 设置界面发送"config_change"事件（包含配置变更）
4. 主控制器接收事件，转发给配置管理器
5. 配置管理器更新内存中的配置
6. 配置管理器验证配置完整性和依赖性
7. 配置管理器发送"config_updated"事件
8. 主控制器通知相关模块（热键监听器、窗口管理器等）
9. 相关模块更新内部状态
10. 配置管理器将配置写入JSON文件
11. 配置管理器创建配置备份（可选）

**异常处理：**
- 配置验证失败：提示用户修正配置
- 文件写入失败：记录错误日志，提示用户检查磁盘空间
- 配置损坏：使用备份恢复

## 四、模块间通信机制

### 4.1 事件总线（Event Bus）

**设计理由：**
- 解耦模块依赖，模块间通过事件通信
- 支持一对多通信（一个事件多个订阅者）
- 便于扩展和维护

**事件分类：**
- **系统事件**：程序启动、关闭、配置变更
- **窗口事件**：窗口显示、隐藏、移动、大小变化
- **输入事件**：热键触发、鼠标点击、键盘输入
- **搜索事件**：搜索请求、搜索结果、搜索取消
- **应用事件**：应用启动、应用添加、应用删除
- **插件事件**：插件加载、插件卸载、插件点击

### 4.2 信号-槽机制

**设计理由：**
- GUI框架原生支持，实现简单
- 支持异步通信，避免阻塞UI线程
- 支持参数传递，便于数据交换

**使用场景：**
- 窗口组件间的通信（搜索框→搜索结果列表）
- 模块间的事件通知（热键监听器→窗口管理器）
- 异步操作的回调（搜索完成→结果更新）

### 4.3 回调函数

**设计理由：**
- 简单直接，适合一对一通信
- 支持动态注册和注销
- 适合简单的数据传递

**使用场景：**
- 搜索引擎结果回调
- 插件接口调用
- 配置变更通知

### 4.4 共享数据

**设计理由：**
- 全局唯一数据源，避免数据不一致
- 减少数据传递开销
- 适合配置等全局数据

**使用场景：**
- 配置数据（通过配置管理器单例）
- 应用列表（通过应用管理器单例）
- 插件清单（通过插件系统单例）

## 五、关键技术决策

### 5.1 Everything集成

**决策：** 直接使用项目内置的es.exe，不依赖系统安装

**理由：**
- 简化部署流程，用户无需额外安装Everything
- 版本可控，避免不同版本兼容性问题
- 搜索性能稳定，依赖官方优化的搜索引擎

**实现要点：**
- es.exe放置在项目everything/目录下
- 搜索时动态调用es.exe，传递搜索参数
- 解析es.exe的JSON输出，转换为内部数据结构
- 支持搜索参数的灵活配置（路径、文件类型、排序等）

**扩展考虑：**
- 支持用户自定义es.exe路径（高级功能）
- 支持多索引文件（不同盘符不同索引）
- 支持正则表达式搜索（高级搜索）

### 5.2 插件架构

**决策：** 预留统一插件接口，支持动态加载和卸载

**理由：**
- 满足用户个性化需求，提升工具扩展性
- 支持第三方开发者贡献插件
- 便于功能模块化开发和维护

**实现要点：**
- 定义插件基类和必须实现的方法
- 使用importlib动态加载插件模块
- 实现插件沙箱机制，隔离插件运行环境
- 提供插件配置界面，支持用户自定义插件参数

**扩展考虑：**
- 支持插件依赖管理（自动安装依赖）
- 支持插件市场（在线发现和安装插件）
- 支持插件间通信（插件协作）

### 5.3 实时搜索

**决策：** 输入关键词时实时触发搜索，无需按回车

**理由：**
- 提升用户体验，减少操作步骤
- 符合现代搜索工具的交互习惯
- 通过防抖机制平衡性能和响应速度

**实现要点：**
- 输入防抖：300ms无输入后触发搜索
- 异步搜索：不阻塞UI线程，保持界面流畅
- 搜索取消：用户连续输入时取消上一次搜索
- 结果缓存：相同关键词复用搜索结果

**扩展考虑：**
- 支持搜索建议（基于搜索历史）
- 支持模糊搜索（容错处理）
- 支持自然语言搜索（未来扩展）

### 5.4 配置持久化

**决策：** 所有配置通过JSON文件保存，重启后恢复

**理由：**
- JSON格式易读易写，便于用户手动编辑
- 便于配置备份和分享
- 解析和序列化性能良好

**实现要点：**
- 配置文件分类存储（settings、apps、plugins）
- 配置变更时实时保存
- 配置验证和默认值处理
- 支持配置导入/导出

**扩展考虑：**
- 支持云同步（不同设备间同步配置）
- 支持多用户配置（不同用户使用不同配置）
- 支持配置版本管理（自动备份历史版本）

## 六、性能优化策略

### 6.1 启动性能

**目标：** 程序启动时间 < 1秒

**优化措施：**
- 延迟加载：非核心模块（插件、历史记录）延迟加载
- 配置缓存：启动时只加载必要配置，其他配置按需加载
- 资源预加载：图标、界面元素预加载到内存

### 6.2 搜索性能

**目标：** 搜索响应时间 < 200ms

**优化措施：**
- 搜索结果缓存（1分钟内相同关键词复用）
- 异步搜索（不阻塞UI线程）
- 搜索取消（避免无效搜索）
- 结果分页（减少单次搜索结果数量）

### 6.3 界面性能

**目标：** 界面响应时间 < 50ms

**优化措施：**
- 窗口复用（显示/隐藏而非重建）
- GPU加速（动画效果使用GPU渲染）
- 虚拟化列表（大量搜索结果时只渲染可见部分）
- 图片缓存（应用图标缓存）

### 6.4 内存占用

**目标：** 常驻内存 < 50MB

**优化措施：**
- 按需加载（插件、历史记录等非核心数据按需加载）
- 资源释放（窗口隐藏时释放非必要资源）
- 对象池（复用常用对象，减少内存分配）

## 七、安全与权限

### 7.1 权限模型

**设计原则：**
- 最小权限原则：只请求必要权限
- 用户授权：敏感操作需要用户明确授权
- 权限隔离：插件运行在受限环境中

**权限分类：**
- **基本权限**：读写配置文件、启动应用
- **高级权限**：文件读写、网络访问、注册表操作
- **系统权限**：全局热键、窗口置顶

### 7.2 安全措施

**插件安全：**
- 插件沙箱机制，限制插件访问系统资源
- 插件权限控制，用户授权后才能访问敏感资源
- 插件签名验证（可选），防止恶意插件

**数据安全：**
- 配置文件加密（可选），防止敏感信息泄露
- 配置备份，防止配置损坏导致数据丢失
- 日志脱敏，避免记录敏感信息

### B. 插件接口定义

#### 插件基类（概念定义）
- `get_icon()`: 返回插件图标（文件路径或资源）
- `handle_click()`: 处理插件点击事件
- `get_info()`: 返回插件信息（名称、版本、作者、描述）
- `on_load()`: 插件加载时回调（可选）
- `on_unload()`: 插件卸载时回调（可选）
- `get_settings()`: 返回插件配置界面（可选）
